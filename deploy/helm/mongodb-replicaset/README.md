# MongoDB Replica Set Helm Chart

Production-grade MongoDB replica set deployment for Control Plane managed resources.

## Overview

This chart deploys a MongoDB replica set using the [Bitnami MongoDB](https://github.com/bitnami/charts/tree/main/bitnami/mongodb) chart as a dependency, with production-ready defaults.

## Features

- **3-node replica set** (configurable)
- **Authentication** enabled by default
- **TLS/SSL** support with auto-generated certificates
- **Persistence** with configurable storage
- **External access** via NodePort or LoadBalancer
- **Prometheus metrics** with ServiceMonitor support
- **Pod Disruption Budget** for high availability
- **Network Policy** support

## Prerequisites

- Kubernetes 1.23+
- Helm 3.8+
- PV provisioner support (for persistence)
- cert-manager (optional, for auto-generated TLS)

## Quick Start

### Install with generated password

```bash
helm install my-mongodb ./mongodb-replicaset \
  --namespace databases --create-namespace
```

### Install with explicit password

```bash
helm install my-mongodb ./mongodb-replicaset \
  --namespace databases --create-namespace \
  --set mongodb.auth.rootPassword=MySecurePassword123
```

### Install with TLS enabled

```bash
helm install my-mongodb ./mongodb-replicaset \
  --namespace databases --create-namespace \
  --set mongodb.auth.rootPassword=MySecurePassword123 \
  --set mongodb.tls.enabled=true \
  --set mongodb.tls.autoGenerated=true
```

### Install with external access

```bash
helm install my-mongodb ./mongodb-replicaset \
  --namespace databases --create-namespace \
  --set mongodb.auth.rootPassword=MySecurePassword123 \
  --set mongodb.externalAccess.enabled=true \
  --set mongodb.externalAccess.service.type=NodePort
```

## Configuration

### Key Parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| `mongodb.architecture` | MongoDB architecture | `replicaset` |
| `mongodb.replicaCount` | Number of replica set members | `3` |
| `mongodb.replicaSetName` | Replica set name | `rs0` |
| `mongodb.auth.enabled` | Enable authentication | `true` |
| `mongodb.auth.rootPassword` | Root user password | `""` (generated) |
| `mongodb.auth.existingSecret` | Existing secret with passwords | `""` |
| `mongodb.tls.enabled` | Enable TLS | `false` |
| `mongodb.tls.autoGenerated` | Auto-generate TLS certs | `true` |
| `mongodb.persistence.enabled` | Enable persistence | `true` |
| `mongodb.persistence.size` | PVC size per replica | `20Gi` |
| `mongodb.externalAccess.enabled` | Enable external access | `false` |
| `mongodb.externalAccess.service.type` | External service type | `NodePort` |
| `mongodb.metrics.enabled` | Enable Prometheus metrics | `false` |

### Example values.yaml for production

```yaml
mongodb:
  architecture: replicaset
  replicaCount: 3
  replicaSetName: rs0
  
  auth:
    enabled: true
    existingSecret: mongodb-credentials
  
  tls:
    enabled: true
    autoGenerated: true
  
  persistence:
    enabled: true
    storageClass: fast-ssd
    size: 100Gi
  
  resources:
    requests:
      cpu: 1000m
      memory: 2Gi
    limits:
      cpu: 4000m
      memory: 8Gi
  
  pdb:
    create: true
    minAvailable: 2
  
  metrics:
    enabled: true
    serviceMonitor:
      enabled: true
```

## Connecting to MongoDB

### From within the cluster

```bash
# Get the password
export MONGODB_ROOT_PASSWORD=$(kubectl get secret -n databases my-mongodb-mongodb -o jsonpath="{.data.mongodb-root-password}" | base64 -d)

# Run a client pod
kubectl run mongodb-client --rm --tty -i --restart='Never' \
  --namespace databases \
  --image docker.io/bitnami/mongodb:7.0 \
  --command -- mongosh "mongodb://my-mongodb-mongodb-0.my-mongodb-mongodb-headless.databases.svc.cluster.local:27017,my-mongodb-mongodb-1.my-mongodb-mongodb-headless.databases.svc.cluster.local:27017,my-mongodb-mongodb-2.my-mongodb-mongodb-headless.databases.svc.cluster.local:27017/?replicaSet=rs0" \
  --username root --password $MONGODB_ROOT_PASSWORD
```

### From your application

Use the connection string from the ConfigMap:

```bash
kubectl get configmap -n databases my-mongodb-replicaset-connection -o jsonpath='{.data.connection-string}'
```

## Backups

For production backups, consider:

1. **MongoDB Cloud Manager** - Official MongoDB backup solution
2. **Percona Backup for MongoDB** - Open source backup tool
3. **Velero with CSI snapshots** - Kubernetes-native backup
4. **Custom CronJob with mongodump** - Simple but effective

Example mongodump CronJob:

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: mongodb-backup
spec:
  schedule: "0 2 * * *"  # Daily at 2 AM
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: backup
            image: bitnami/mongodb:7.0
            command:
            - /bin/bash
            - -c
            - |
              mongodump --uri="$MONGODB_URI" --archive=/backup/$(date +%Y%m%d).archive --gzip
            env:
            - name: MONGODB_URI
              valueFrom:
                secretKeyRef:
                  name: mongodb-backup-uri
                  key: uri
            volumeMounts:
            - name: backup
              mountPath: /backup
          restartPolicy: OnFailure
          volumes:
          - name: backup
            persistentVolumeClaim:
              claimName: mongodb-backups
```

## Upgrading

```bash
helm upgrade my-mongodb ./mongodb-replicaset \
  --namespace databases \
  --reuse-values \
  --set mongodb.image.tag=7.0.5
```

## Uninstalling

```bash
# Uninstall release (keeps PVCs)
helm uninstall my-mongodb -n databases

# Delete PVCs if you want to remove data
kubectl delete pvc -n databases -l app.kubernetes.io/instance=my-mongodb
```

## Control Plane Integration

This chart creates a ConfigMap (`<release>-connection`) with connection details that the Control Plane API can discover and display to users. The ConfigMap includes:

- Connection string
- Service names
- TLS status
- External access info

The Control Plane UI reads this ConfigMap to show connection details in the Resources page.
